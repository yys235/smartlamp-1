# Smart Lamp 灯泡控制功能点拆分

## 1. 目标范围

本文只拆分“灯泡控制”相关功能，不展开项目的构建、部署、事件框架和通用返回结构。

拆分依据来自当前代码实现，覆盖两部分：

- Java 服务端对灯泡的控制能力
- Arduino/ESP8266 固件对灯泡的控制能力

## 2. 灯泡控制主流程

从代码现状看，灯泡控制链路可以拆为 4 个阶段：

1. 发现网关
2. 获取灯泡列表
3. 修改目标灯泡状态
4. 下发更新并等待状态回读

也就是说，当前系统不是“直接控制单灯”，而是先拉取整组灯状态，再修改内存中的目标灯，最后把整组灯状态批量写回网关。

## 3. Java 服务端控制功能点

### 3.1 灯泡发现与可控状态建立

功能点：

- 自动监听网关 UDP 广播
- 自动记录网关 IP
- 自动记录 `gatewayId`
- 网关连接成功后进入可控制状态

输入：

- 局域网内网关广播包

输出：

- 服务端拿到后续控制所需的网关地址和标识

限制：

- 必须与灯泡网关处于同一局域网
- 在未发现网关前，无法执行实际控制

### 3.2 灯泡列表读取

功能点：

- 从网关读取当前所有灯泡
- 解析每个灯泡的 `deviceId`、RGB、亮度
- 在控制前把灯泡列表封装为 `Collection`

输入：

- 网关 TCP 响应数据

输出：

- `Lamp[]`
- `Collection`

用途：

- 为后续全灯控制、单灯控制、状态查询提供基础数据

### 3.3 全灯打开

功能点：

- 当 `/on` 未传 `deviceId` 或传 `0` 时，对全部灯泡生效
- 支持同时设置全灯颜色和亮度

可设置项：

- `intensity`
- `r`
- `g`
- `b`

默认行为：

- 未传颜色时，默认白光
- 未传亮度时，默认 `255`

业务效果：

- 所有灯泡统一改成同一颜色和亮度

### 3.4 单灯打开

功能点：

- `/on?deviceId=xxx` 只对指定灯泡生效

匹配逻辑：

- 遍历灯泡列表
- 用 `deviceId.equals(lamp.getId())` 匹配目标灯泡

可设置项：

- `intensity`
- `r`
- `g`
- `b`

业务效果：

- 指定灯泡被设置为目标亮度和颜色
- 其他灯泡保持原状态

### 3.5 全灯关闭

功能点：

- 当 `/off` 未传 `deviceId` 或传 `0` 时，对所有灯泡执行关闭

关闭方式：

- 把每个灯泡的 `intensity` 设置为 `0`

业务效果：

- 全部灯泡灭灯
- 原有 RGB 值保留在对象中，但亮度为 0

### 3.6 单灯关闭

功能点：

- `/off?deviceId=xxx` 仅关闭指定灯泡

关闭方式：

- 匹配指定 `deviceId`
- 仅将该灯的 `intensity` 改为 `0`

业务效果：

- 只关闭一盏灯
- 其他灯不受影响

### 3.7 亮度控制

功能点：

- 通过 `/on` 的 `intensity` 参数设置亮度
- 通过 `/off` 将亮度直接置为 `0`

当前能力边界：

- 亮度是直接赋值，没有做区间校验
- 没有单独的 `/brightness` 接口

说明：

- 当前亮度控制本质上是灯泡更新报文中的一个字段，不是单独协议能力封装

### 3.8 颜色控制

功能点：

- 通过 `/on` 的 `r/g/b` 参数设置颜色

当前能力边界：

- 支持自定义 RGB 数值
- 没有颜色名称接口
- 没有预设场景色接口

说明：

- 颜色控制和开灯动作绑定在一起
- 当前没有单独“改色但不改亮度”的 HTTP 接口

### 3.9 状态查询

功能点：

- 查询当前灯泡列表
- 查询是否全灯关闭
- 查询最近一次通信时间

接口：

- `GET /status`

返回的灯控相关字段：

- `allOff`
- `lastCommunication`
- `lamps`

用途：

- 给上层系统做页面展示
- 给调用方确认控制结果

### 3.10 灯泡状态确认与补发

功能点：

- 控制请求发出后等待 5 秒
- 再次获取灯泡列表
- 对比目标状态与回读状态
- 不一致时重发一次更新指令

价值：

- 降低一次写入失败造成的状态不一致

边界：

- 只重试 1 次
- 没有失败结果返回给 HTTP 调用方

### 3.11 初次接入初始化

功能点：

- 网关首次连接成功时，执行一次灯泡初始化操作

流程：

1. 读取当前灯泡列表
2. 暂时把所有灯改成满亮白光
3. 再恢复原灯态

作用判断：

- 更偏向通信确认和设备唤醒
- 不是常规业务控制能力

### 3.12 灯泡命名映射预留

功能点：

- `LampStore` 预设了灯泡名称与 `deviceId` 的映射关系

当前已有映射：

- `front_left`
- `front_right`
- `back_left`
- `back_right`

当前状态：

- 仅有数据预置
- 还没有接入 HTTP 控制逻辑

这意味着系统“具备命名控制扩展基础”，但当前并未真正提供“按名称控制灯泡”的能力。

## 4. Arduino/ESP8266 侧控制功能点

### 4.1 灯泡状态同步

功能点：

- 首次发现网关后自动读取全部灯泡状态
- 检测到设备变化后重新读取全部灯泡状态
- 将结果缓存到 `lampArray`

用途：

- 给语音控制和本地按钮控制提供最新设备状态

### 4.2 全灯开关切换

功能点：

- `switchAllLamps()` 实现全灯开关切换

规则：

- 当前亮度大于 0 的灯，切换为关闭
- 当前亮度等于 0 的灯，切换为 `0x28`

触发入口：

- Blinker 按钮 `Button1`
- 天猫精灵电源控制回调

### 4.3 全灯批量下发

功能点：

- 把缓存中的所有有效灯泡组装成批量更新报文
- 一次性写回网关

用途：

- 所有 Arduino 侧灯控动作最终都通过该函数生效

### 4.4 语音开关控制

功能点：

- 接收天猫精灵开灯/关灯指令
- 触发本地全灯切换逻辑

当前实现特点：

- 实际不是显式设置“开”或“关”
- 而是通过 `switchAllLamps()` 做全灯翻转

影响：

- 如果当前灯态与语音平台认知不一致，可能出现反向切换

### 4.5 语音颜色控制

功能点：

- 支持语音设置颜色
- 将颜色名称映射为 RGB
- 修改 `lampArray[0]` 的颜色
- 下发到网关

当前支持颜色：

- Red
- Yellow
- Blue
- Green
- White
- Black
- Cyan
- Purple
- Orange

当前实现边界：

- 实际只修改 `lampArray[0]`
- 没有把颜色同步应用到所有灯泡

### 4.6 语音亮度控制

功能点：

- 支持设置绝对亮度
- 支持设置相对亮度

实现方式：

- 只修改 `lampArray[0].intensity`
- 然后调用 `updateAllLamps()`

当前边界：

- 主要以第一个灯泡作为语音控制代表
- 没有全灯亮度同步逻辑

### 4.7 语音状态查询

功能点：

- 支持天猫精灵查询开关状态
- 支持查询颜色
- 支持查询模式
- 支持查询色温
- 支持查询亮度

数据来源：

- 主要读取 `lampArray[0]`

说明：

- 当前语音平台看到的灯状态，本质上是第一个灯泡的状态映射

### 4.8 色温与模式接口占位

功能点：

- 已接入模式设置回调
- 已接入取消模式回调
- 已接入色温设置回调
- 已接入相对色温设置回调

当前状态：

- 只响应语音平台
- 没有映射到底层灯泡控制协议

因此这些能力应视为“接口已留出，控制未落地”。

## 5. 按控制维度拆分

### 5.1 按控制对象拆分

已实现：

- 全部灯泡控制
- 指定 `deviceId` 的单灯控制

预留未接入：

- 按灯泡名称控制

### 5.2 按控制动作拆分

已实现：

- 查询灯泡列表
- 开灯
- 关灯
- 设置亮度
- 设置 RGB 颜色
- 查询控制结果

部分实现：

- 语音模式控制
- 语音色温控制

未实现：

- 定时控制
- 渐变效果
- 场景模式
- 分组控制
- 命名控制入口

### 5.3 按触发入口拆分

已实现：

- HTTP 接口控制
- Blinker 按钮控制
- 天猫精灵语音控制

### 5.4 按控制粒度拆分

Java 服务端：

- 支持全灯
- 支持单灯

Arduino 固件：

- 全灯切换支持
- 语音颜色和亮度主要围绕第一个灯泡

## 6. 当前灯控能力总结

如果只看“灯泡控制”本身，当前仓库已经具备的核心能力是：

- 自动发现灯泡网关
- 获取全部灯泡状态
- 通过 HTTP 控制全灯或单灯开关
- 通过 HTTP 设置灯泡 RGB 和亮度
- 通过 Arduino/Blinker 实现全灯切换
- 通过天猫精灵实现基础的开关、颜色、亮度交互

当前最明显的控制边界是：

- Java 侧没有命名灯控制入口
- Java 侧缺少独立亮度接口、独立颜色接口和分组控制
- Arduino 侧语音控制主要基于第一个灯泡，不是严格的多灯一致控制
- 语音模式和色温仅完成平台接线，未完成真实灯控落地
