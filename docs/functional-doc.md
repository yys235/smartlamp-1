# Smart Lamp 功能文档

## 1. 项目概述

Smart Lamp 是一个围绕局域网智能灯泡构建的双端项目，包含：

- Java 服务端：负责发现灯泡网关、查询灯泡状态、通过 HTTP 接口对灯泡下发控制指令。
- Arduino/ESP8266 固件：负责把局域网灯泡协议接入 Blinker 与天猫精灵，实现语音和按键控制。

当前代码明确支持的灯泡协议对象是 README 中提到的 Keelight 与乐逗智能灯泡。

## 2. 系统组成

### 2.1 Java 服务端

技术栈：

- Java 11
- Spring Boot 2.5.2
- Lombok

核心职责：

- 监听网关 UDP 广播
- 识别网关地址与 `gatewayId`
- 通过 TCP 协议拉取灯泡列表
- 通过 TCP 协议批量更新灯泡状态
- 暴露简单 HTTP 控制接口

### 2.2 Arduino/ESP8266 固件

技术栈与依赖：

- ESP8266 Wi-Fi
- WiFiUDP / WiFiClient
- Blinker SDK
- AliGenie Light 能力接口

核心职责：

- 连接 Wi-Fi 与 Blinker 平台
- 接收网关广播并建立灯泡控制链路
- 查询和缓存局域网内灯泡状态
- 将语音平台指令转换为灯泡协议指令

## 3. 核心功能清单

### 3.1 网关自动发现

Java 服务端和 Arduino 固件都在本地监听 UDP `41328` 端口。

已实现能力：

- 自动接收网关广播包
- 从广播包中解析网关 IP 地址
- 从广播包第 2 到 5 字节解析 `gatewayId`
- 从广播包第 21、22 字节判断设备是否发生增减

触发结果：

- 首次发现网关时触发“网关连接成功”逻辑
- 检测到设备数量变化时触发“灯泡状态变化”逻辑

### 3.2 灯泡列表读取

系统通过 TCP `41330` 端口向网关发送“获取灯泡列表”请求，返回结果被解析为灯泡数组。

单个灯泡当前包含以下属性：

- `deviceId`
- `red`
- `green`
- `blue`
- `intensity`

Java 侧会把响应每 8 个字节解析为一个灯泡对象。

### 3.3 灯泡开关控制

Java 服务端已提供按设备或全量控制开关灯能力：

- `/on`：支持全灯打开或指定灯打开
- `/off`：支持全灯关闭或指定灯关闭

控制方式：

- 全量控制：遍历所有灯泡并修改目标状态
- 单灯控制：按 `deviceId` 匹配目标灯泡并修改状态

### 3.4 亮度与颜色控制

Java 服务端支持在 `/on` 接口中同时设置：

- 亮度 `intensity`
- 红色通道 `r`
- 绿色通道 `g`
- 蓝色通道 `b`

默认值：

- `intensity` 默认为 `255`
- `r/g/b` 默认为 `255`

这意味着未传颜色参数时，默认会以白光满亮度打开灯泡。

### 3.5 灯泡状态初始化

Java 服务端在首次连接网关后会执行一次初始化动作：

1. 获取当前灯泡列表
2. 暂时把所有灯设置为 `RGBI(255,255,255,255)`
3. 再恢复为原始状态

该逻辑的作用更像“激活和确认通信链路”，不是业务层的常规开灯流程。

### 3.6 状态确认与重试

Java 服务端在更新灯泡后会执行一次弱确认逻辑：

1. 发送更新请求
2. 等待 5 秒
3. 再次读取灯泡列表并与目标状态对比
4. 如不一致，则重发一次更新请求

此外，客户端还做了 `200ms` 的请求节流，避免过快连续访问网关。

### 3.7 状态查询接口

Java 服务端提供 `/status` 接口返回系统状态。

当前返回内容包括：

- 网关状态字符串 `state`
- 是否全部关闭 `allOff`
- 最后一次与网关通信时间 `lastCommunication`
- 当前灯泡列表 `lamps`

## 4. Java 服务端模块说明

### 4.1 入口模块

`SmartLampApplication` 为标准 Spring Boot 启动入口，仅负责启动容器。

### 4.2 控制器模块

`LampController` 对外暴露 3 个 HTTP 接口：

#### `GET /status`

功能：

- 返回当前连接状态
- 返回全灯是否关闭
- 返回最后通信时间
- 返回当前灯泡列表

#### `GET /on`

参数：

- `deviceId`：可选，未传或传 `0` 时表示全量控制
- `intensity`：可选，亮度
- `r`：可选，红色值
- `g`：可选，绿色值
- `b`：可选，蓝色值

功能：

- 打开全部灯泡或指定灯泡
- 支持同时设定颜色与亮度

#### `GET /off`

参数：

- `deviceId`：可选，未传或传 `0` 时表示全量控制

功能：

- 把全部灯泡或指定灯泡的亮度设置为 `0`

### 4.3 控制协调模块

`Controller` 是 Java 服务端的业务协调层，负责：

- 监听网关连接事件
- 监听灯泡状态变化事件
- 对 `Client` 提供简化封装
- 提供全灯是否关闭判断
- 暴露最近通信时间

对外方法：

- `getLamps()`
- `updateLamps()`
- `areAllLightsOff()`
- `isConnected()`
- `getLastCommunication()`

### 4.4 网络客户端模块

`Client` 是与灯泡网关直接通信的核心模块。

主要能力：

- 订阅 UDP 广播事件
- 保存网关 IP
- 保存网关 `gatewayId`
- 读取灯泡列表
- 发送灯泡更新请求
- 记录最近一次通信时间

关键实现点：

- UDP 用于发现网关
- TCP 用于收发实际控制指令
- 请求之间最少间隔 `200ms`

### 4.5 UDP 监听模块

`UdpClient` 负责：

- 打开 UDP `41328` 端口
- 启动监听线程
- 接收到广播后触发 `UdpPacketReceivedEvent`

### 4.6 事件模块

事件系统由 `EventManager` 与 `EventHandler` 组成，当前支持 4 类事件：

- `GatewayConnectedEvent`
- `LampStateChangedEvent`
- `UdpPacketReceivedEvent`
- `ExceptionEvent`

事件特点：

- 监听器可重复注册防重
- 每次触发事件时，使用新线程异步回调监听器

### 4.7 数据模型

#### `Lamp`

表示单个灯泡，提供：

- 颜色与亮度读写
- `setRGB()`
- `setRGBI()`
- 转换为协议字节数组 `getBytes()`

#### `Collection`

表示灯泡数组包装器，提供：

- 迭代能力
- 拷贝能力
- 长度读取
- 原始数组访问
- 集合对比

### 4.8 请求封装

当前定义了两类底层协议请求：

- `GetLampsRequest`
- `UpdateLampsRequest`

作用：

- 生成灯泡网关协议所需的二进制报文
- 保存请求后的响应数据

## 5. Arduino/ESP8266 固件功能说明

### 5.1 基础连接能力

固件启动后会：

1. 初始化串口日志
2. 初始化 Blinker
3. 绑定调试按钮和数据回调
4. 绑定天猫精灵灯光控制回调
5. 开启 UDP 监听

需要用户手工配置：

- `auth`
- `ssid`
- `pswd`

### 5.2 局域网灯泡发现与同步

固件在 `loop()` 中持续执行：

- 运行 `Blinker.run()`
- 检查是否收到 UDP 广播
- 解析网关地址与网关 ID
- 首次连接时自动拉取所有灯泡
- 检测到设备增减时重新拉取所有灯泡

### 5.3 灯泡列表缓存

固件使用 `lampArray[10]` 缓存最多 10 个灯泡。

`saveLamps()` 的职责：

- 按协议解析灯泡列表
- 保存到本地数组
- 输出串口调试信息

### 5.4 本地调试按钮

已定义两个 Blinker 按钮：

- `Button1`：调用 `switchAllLamps()`，切换全灯开关
- `Button2`：调用 `getAllLamps()`，重新查询全部灯泡

### 5.5 全灯切换

`switchAllLamps()` 会遍历当前缓存的灯泡：

- 如果亮度大于 0，则设为 0
- 否则设为 `0x28`

随后调用 `updateAllLamps()` 下发更新。

### 5.6 灯泡批量更新

`updateAllLamps()` 会：

- 统计当前有效灯泡数量
- 组装批量更新二进制协议
- 通过 TCP `41330` 发送给网关

### 5.7 天猫精灵控制能力

当前已接入以下 AliGenie 控制回调：

- 电源状态控制
- 颜色控制
- 模式控制
- 取消模式
- 亮度控制
- 相对亮度控制
- 色温查询/设置回调
- 状态查询

#### 已真正作用到灯泡协议的能力

- 开关
- 颜色
- 亮度
- 相对亮度
- 状态查询

#### 当前仅对平台响应、未真正控制灯泡协议的能力

- 模式切换
- 取消模式
- 色温设置
- 相对色温设置

也就是说，上述能力会向天猫精灵返回响应，但不会把对应效果映射成实际灯泡状态。

### 5.8 颜色映射

固件已实现若干标准颜色名称与 RGB 的双向映射，包括：

- Red
- Yellow
- Blue
- Green
- White
- Black
- Cyan
- Purple
- Orange

## 6. 对外接口说明

### 6.1 HTTP 接口

#### `/status`

请求方式：

- `GET`

返回示例结构：

```json
{
  "code": "0000",
  "msg": "success",
  "data": {
    "state": "CONNECTED",
    "allOff": false,
    "lastCommunication": "12:34:56.789",
    "lamps": [
      {
        "id": 123456789,
        "red": 255,
        "green": 255,
        "blue": 255,
        "intensity": 255
      }
    ]
  }
}
```

说明：

- `state` 在当前实现中固定返回 `CONNECTED`
- `lamps` 只有在成功获取到灯泡列表时才会返回

#### `/on`

请求方式：

- `GET`

参数：

- `deviceId`
- `intensity`
- `r`
- `g`
- `b`

示例：

```text
/on
/on?deviceId=1680764563
/on?deviceId=1680764563&intensity=128&r=255&g=128&b=0
```

#### `/off`

请求方式：

- `GET`

参数：

- `deviceId`

示例：

```text
/off
/off?deviceId=1680764563
```

### 6.2 局域网协议与端口

- UDP `41328`：监听网关广播
- TCP `41330`：查询与更新灯泡状态

协议特征：

- `GetLampsRequest` 生成长度为 15 字节的查询报文
- `UpdateLampsRequest` 根据灯泡数量生成批量更新报文

## 7. 启动与运行方式

### 7.1 Java 服务端

运行前提：

- JDK 11
- Maven 环境
- 与智能灯泡网关位于同一局域网

服务端启动后会自动开始监听网关广播，无需额外配置文件参数。

### 7.2 Arduino/ESP8266 固件

运行前提：

- Arduino IDE 或兼容构建环境
- ESP8266 开发板
- 已填写 Blinker 设备密钥和 Wi-Fi 配置

固件上传完成后，即可通过 Blinker 或天猫精灵调用对应控制能力。

## 8. 当前实现边界与注意事项

以下内容是代码现状，不是理论能力：

- HTTP 接口没有鉴权，适合内网场景，不适合直接暴露公网。
- `LampStore` 中存在预设灯泡名称映射，但当前代码没有真正使用这些映射。
- `/status` 接口中的 `state` 固定写死为 `CONNECTED`，没有根据 `Controller.isConnected()` 动态返回。
- Java 服务端没有持久化层，灯泡状态完全依赖实时网关查询。
- Java 服务端异常主要写日志，没有统一错误恢复策略。
- Arduino 固件的模式和色温接口尚未映射到底层灯泡协议。
- Arduino 固件默认缓存上限为 10 个灯泡。
- 当前仓库测试代码只有 Spring Boot 启动测试，业务行为没有自动化测试覆盖。

## 9. 适用场景

当前实现更适合以下场景：

- 家庭或办公室局域网中的智能灯泡统一控制
- 将原本仅支持专有 App 的灯泡接入语音平台
- 在 NAS、软路由或轻量服务器上部署灯泡控制服务
- 基于现有协议继续扩展到其他语音平台或智能家居平台

## 10. 建议后续演进方向

如果继续迭代，优先级较高的方向包括：

- 为 HTTP 接口增加鉴权和错误码
- 补齐动态连接状态判断
- 将 `LampStore` 真正接入命名灯控制
- 为 Arduino 侧补齐模式和色温的真实控制映射
- 增加自动化测试和协议说明文档
- 将关键参数改为配置项而不是硬编码
